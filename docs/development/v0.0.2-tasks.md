## Overview

This phase implements the missing UI/UX features that make data-assistant's interface engaging and professional. We'll adapt the SQL assistant's UX patterns to the legal domain.

**Key Design Principles**:
- ✅ Keep users engaged with visible thinking/tool calls
- ✅ Centered, welcoming first-message experience
- ✅ Persian-first RTL design
- ✅ Authentication required for per-user conversation history (added in 11.2)
- ✅ Keep Arize Phoenix for observability (no Langfuse migration)
- ✅ No user settings UI (admin-controlled via config.yaml)

**Key Learnings from 11.2** (critical for future tasks):
- Chainlit tables use **camelCase** columns (`createdAt`, `userId`), not snake_case
- `createdAt` stored as **TEXT** (ISO string), not TIMESTAMP
- Never override `execute_sql()` — let parent class handle it
- `@cl.password_auth_callback` required for per-user sidebar; JWT secret in `.env`
- Translation file: `.chainlit/translations/fa-IR.json`, set in `.chainlit/config.toml`

**Key Learnings from 11.3** (critical for future tasks):
- `@cl.step` on a **sync function** does NOT reliably render in UI — always use `async with cl.Step():`
- Send `cl.Message()` **AFTER** `agent.run()` so steps appear before the final answer
- Update `step.name` inside the context manager to show result-based labels ("N سند پیدا شد")
- `steps` table needs `"defaultOpen"` BOOLEAN column — Chainlit always sends it
- Set `messages.status.used = ""` in translation to hide "استفاده شد" suffix
- Our LiteLLM model emits only `ToolCallPart` — no `TextPart` or `ThinkingPart` alongside tools
- Translation files live in `.chainlit/translations/fa-IR.json`; language set in `config.toml`

```python
from pydantic_ai.agent import CallToolsNode
from pydantic_ai.messages import ThinkingPart, TextPart, ToolCallPart

async def run(self, user_query: str, ...):
    async with self.agent.iter(user_prompt=user_query, ...) as run:
        async for node in run:
            if isinstance(node, CallToolsNode):
                # Extract thinking parts
                thinking_parts = []
                text_parts = []
                has_tools = False

                for part in node.model_response.parts:
                    if isinstance(part, ToolCallPart):
                        has_tools = True
                    if isinstance(part, ThinkingPart):
                        thinking_parts.append(part.content)
                    if isinstance(part, TextPart):
                        text_parts.append(part.content)

                # Show thinking
                if thinking_parts:
                    await show_thinking("\n\n".join(thinking_parts))
                elif text_parts and has_tools:
                    # Some models use text as "faked" thinking
                    await show_thinking("\n\n".join(text_parts))

    return run.result.output, run.result.all_messages()
```

```python
import chainlit as cl

@cl.step(name="جستجوی اسناد حقوقی", type="retrieval", show_input=True)
async def search_documents(
    query: str,
    tags: list[str] | None = None,
    doc_types: list[str] | None = None,
    limit: int = 20
) -> list[DocSummary]:
    """Search legal documents using full-text search."""
    current_step = cl.context.current_step

    # Set input
    input_text = f"**Query**: {query}\n"
    if tags:
        input_text += f"**Tags**: {', '.join(tags)}\n"
    if doc_types:
        input_text += f"**Document Types**: {', '.join(doc_types)}\n"
    input_text += f"**Limit**: {limit}"
    current_step.input = input_text

    # Perform search
    results = await _search_documents_impl(query, tags, doc_types, limit)

    # Set output (summary)
    if results:
        output_text = f"**پیدا شد {len(results)} سند**\n\n"
        for i, doc in enumerate(results[:5], 1):  # Show top 5
            output_text += f"{i}. {doc.title} (امتیاز: {doc.score:.2f})\n"
        if len(results) > 5:
            output_text += f"\n... و {len(results) - 5} سند دیگر"
    else:
        output_text = "هیچ سندی پیدا نشد"

    current_step.output = output_text
    return results
```

```javascript
// ==================== Notification System ====================

// Request permission on page load
function requestNotificationPermission() {
  if (!('Notification' in window)) {
    console.warn('This browser does not support notifications');
    return;
  }
  if (Notification.permission === 'default') {
    Notification.requestPermission();
  }
}

// Call on load
window.addEventListener('load', () => {
  setTimeout(requestNotificationPermission, 1000);
});

// Tab title flashing for notifications
const TabNotification = {
  originalTitle: document.title,
  interval: null,
  isFlashing: false,

  start(message) {
    if (this.isFlashing) return;

    this.originalTitle = document.title;
    this.isFlashing = true;
    let flashState = false;

    const shortMessage = message.length > 30 ? message.substring(0, 30) + '...' : message;

    this.interval = setInterval(() => {
      if (document.hidden) {
        document.title = flashState ? this.originalTitle : shortMessage;
        flashState = !flashState;
      } else {
        this.stop();
      }
    }, 1000);
  },

  stop() {
    if (this.interval) {
      clearInterval(this.interval);
      this.interval = null;
    }
    if (this.isFlashing) {
      document.title = this.originalTitle;
      this.isFlashing = false;
    }
  }
};

// Stop flashing when tab becomes visible
document.addEventListener('visibilitychange', () => {
  if (!document.hidden && TabNotification.isFlashing) {
    TabNotification.stop();
  }
});

// Send desktop notification
function sendNotification(title, body, icon, tag) {
  if (!('Notification' in window)) {
    return false;
  }

  if (document.hidden && Notification.permission === 'granted') {
    // Start tab title flashing
    TabNotification.start(body || title);

    // Create notification
    const notification = new Notification(title, {
      body: body,
      icon: icon || '/public/law_icon.svg',
      tag: tag || 'law-agent'
    });

    notification.onclick = function () {
      window.focus();
      notification.close();
      TabNotification.stop();
    };

    return true;
  }
  return false;
}

// Chainlit callback integration
window.addEventListener("chainlit-call-fn", (e) => {
  const {name, args, callback} = e.detail;
  if (name === "send-notification") {
    callback(sendNotification(args.title, args.body, args.icon, args.tag));
  } else {
    callback(`Unsupported function '${name}'`);
  }
});
```

```javascript
function addCopyButtons() {
  document.querySelectorAll('pre code').forEach(block => {
    // Skip if already has button
    if (block.parentElement.querySelector('.copy-button')) return;

    // Create button
    const button = document.createElement('button');
    button.className = 'copy-button';
    button.textContent = 'کپی';
    button.title = 'کپی کد';

    button.onclick = () => {
      navigator.clipboard.writeText(block.textContent).then(() => {

        setTimeout(() => button.textContent = 'کپی', 2000);
      });
    };

    // Position button
    const pre = block.parentElement;
    pre.style.position = 'relative';
    pre.insertBefore(button, block);
  });
}
```
